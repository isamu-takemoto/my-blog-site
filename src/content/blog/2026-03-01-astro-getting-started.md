ずっと「ブログ書きたいな」と思いながら後回しにしてたんですが、ようやく重い腰を上げて作りました。

正直に言うと、このブログの構築はほぼ Claude Code（CC）と一緒にやりました。デザインの細かい調整からコンポーネントの実装まで、「こうしたい」を伝えながら進めていく感じ。自分で全部書くよりずっと速いし、知らなかった書き方を学べる場面もあって、思ってたよりいい体験でした。AI と一緒に作るのが普通になってきてる実感がある。

## なぜ Astro？

Next.js は触ったことがある程度なんですが、ブログに使うにはちょっとオーバーキルかなと。ただ記事を表示したいだけなのに、なんかいろいろ設定多いんですよね。

Astro はコンテンツサイト向けに特化されてて、デフォルトで JavaScript をほぼ出力しない。つまりめちゃくちゃ速い。それだけで採用理由として十分でした。

## セットアップはこれだけ

```bash
npm create astro@latest . -- --template blog
```

テンプレートを使えば MDX サポートやサイトマップ生成が最初から入った状態で始められます。ここからカスタマイズしていく感じ。

## 記事の管理

`src/content/blog/` に Markdown ファイルを置くだけ。frontmatter にタイトルと日付を書けば記事として認識されます。

```markdown
---
title: 記事タイトル
pubDate: 2026-03-01
tags:
  - Astro
---

ここに本文を書く
```

シンプルでいい。

## デプロイは Vercel に投げる

GitHub リポジトリを Vercel に連携するだけで完了。push するたびに勝手にビルドして反映してくれます。自前でサーバー管理しなくていいのは本当にありがたい。

## 作ってみて

思ってたより全然ラクでした。デザインをゼロから作るのが一番時間かかったくらいで、Astro 自体は詰まるところがほぼなかった。

ブログ自体を作るのが目的になりかけてたので、そろそろちゃんと記事も書いていきたいと思います。
